PROMPT 1 - SINGLE PASS (FULL BUILD IN ONE GO)
================================================================
You are implementing a Python program named CallGhost.py.

Goal:
Build a local speech-to-text utility with two modes:
- file mode: transcribe an input audio file to TXT + SRT
- live mode: capture from an input audio device, support chapter-based recording, and generate TXT + SRT + final audio per chapter

Core requirements:
1. CLI:
- Use argparse with subcommands: file, live
- Support --config config.ini for defaults; CLI flags override config values
- If no args are provided, run interactive startup:
  - list available input devices
  - monitor 3 seconds and show simple activity indicator (Active/Silent)
  - ask user to choose device index

2. File mode:
- Input: audio path
- Output: txt + srt files
- Use Whisper local model
- Handle missing input, missing ffmpeg, and path errors with actionable messages

3. Live mode:
- Capture input audio and manage chapter flow:
  - Start chapter 001 automatically
  - SPACE closes current chapter and triggers background processing
  - SPACE again (only after previous processing is complete) starts next chapter (002, 003, ...)
  - ESC closes active chapter (if any), processes it, then exits cleanly
  - If silence lasts >= stop_when_silent_for: beep 3 times, auto-close chapter, process it, then wait for SPACE/ESC
- For each chapter generate:
  - {prefix}_NNN.txt
  - {prefix}_NNN.srt
  - {prefix}_NNN.ogg (or selected format)
- Keep processing sequential (one chapter processing thread at a time)

4. Audio + transcription behavior:
- Capture as mono 16k for Whisper compatibility
- Keep robust formatting for SRT timestamps
- Preserve/implement overlap-aware deduplication logic where needed

5. Config:
- Auto-create config.ini with commented defaults if missing
- Include app/capture/audio/output sections

6. Docs + quality:
- Keep code readable and typed
- Add clear docstrings and concise comments for non-obvious logic
- Update README.md in English with:
  - setup
  - usage
  - chapter controls
  - practical use cases
  - responsible use
  - CPU vs CUDA guidance
  - lessons learned and prompt-writing tips

7. Validation:
- Run python -m py_compile CallGhost.py
- Show brief summary of files changed and key behaviors implemented

Constraints:
- Do not add unnecessary external dependencies
- Do not perform broad refactors outside required scope
- Keep behavior predictable and user-facing messages professional

PROMPT 4 - MULTI-STEP MODE: STEP 3 (LIVE CHAPTER MODE + THREADING)
================================================================
Implement chapter-based live mode:

- Auto-start chapter 001
- SPACE:
  - if recording: close chapter and process in background
  - if idle and processing done: start next chapter
  - if processing not done: block SPACE and show status message
- ESC:
  - close active chapter if any
  - process pending chapter
  - exit cleanly
- Silence timeout:
  - beep 3 times
  - close chapter
  - process it
  - remain waiting for SPACE/ESC

Output naming:
- {prefix}_001, {prefix}_002...
- write txt, srt, final audio per chapter

Processing constraint:
- one chapter processing worker at a time

After implementation:
- run python -m py_compile CallGhost.py
- provide short explanation of chapter state machine.
